# 📝 순차적 과제 시스템 테스트 가이드

## ✅ 적용 완료 사항

### 1. 캐시 방지 메타 태그 추가
- 브라우저가 항상 최신 버전을 로드하도록 설정
- `lesson.html` 파일에 적용 완료

### 2. 디버깅 로그 추가
- 콘솔에서 과제 시스템 로드 확인 가능
- 버전 정보 표시: `2024-12-19 순차적 잠금 해제`

### 3. 순차적 과제 시스템
- 퀴즈 → 카톡 → 블로그 → 사진 → 체크리스트 순서
- 이전 과제 완료 시 다음 과제 자동 활성화
- 완료 상태 실시간 표시 (✓ 체크마크)
- 잠금 상태 표시 (🔒 아이콘 + 흐림 효과)

---

## 🧪 테스트 방법

### Step 1: 브라우저 캐시 완전 삭제

#### Chrome/Edge
1. **Ctrl + Shift + Delete** 키를 누름
2. **"캐시된 이미지 및 파일"** 체크
3. **"전체 기간"** 선택
4. **"데이터 삭제"** 클릭

#### 또는 시크릿 모드 사용
- **Ctrl + Shift + N** (시크릿 창)

---

### Step 2: lesson.html 열기

**정확한 경로로 열기:**
```
C:\Users\이현수\Documents\GitHub\Allround-English\lesson.html?lessonId=YOUR_LESSON_ID
```

**또는 Explorer에서:**
1. `C:\Users\이현수\Documents\GitHub\Allround-English` 폴더 열기
2. `lesson.html` 우클릭
3. "연결 프로그램" → 브라우저 선택

---

### Step 3: 개발자 도구로 확인

#### F12 키를 눌러 개발자 도구 열기

#### Console 탭에서 확인할 내용:

**✅ 정상 로딩 시 나타나는 메시지:**
```
🎯 [과제 시스템] 버전: 2024-12-19 순차적 잠금 해제
🎯 [과제 시스템] assignmentProgress: {quiz: {...}, kakao: {...}, ...}
🎯 [과제 시스템] renderAssignmentCards: function
```

**❌ 오류가 있다면:**
- 빨간색 에러 메시지 확인
- 스크린샷 찍어서 공유

---

### Step 4: 과제 순차 시스템 테스트

#### 4-1. 강의 90% 이상 시청
- 영상을 90% 이상 시청하면 과제 섹션이 나타남

#### 4-2. 첫 번째 과제(퀴즈) 확인
- ✅ 퀴즈 카드만 밝게 표시
- 🔒 나머지 카드들은 흐릿하고 잠금 표시

#### 4-3. 퀴즈 완료
1. 퀴즈 카드 클릭
2. 문제 풀기
3. 제출

**기대 결과:**
- ✓ 퀴즈 카드에 체크마크
- 💬 카톡 카드 활성화 (밝아짐)

#### 4-4. 순차적으로 완료
1. 카톡 링크 열기 → 블로그 활성화
2. 블로그 링크 열기 → 사진 활성화
3. 사진 제출 → 체크리스트 활성화
4. 체크리스트 완료 → 모든 과제 완료! 🎉

---

## 🔍 문제 해결

### 문제: 과제 섹션이 안 보여요
**해결:**
- 영상을 90% 이상 시청했는지 확인
- 또는 URL에 `?directSubmit=true` 추가

### 문제: 여전히 이전 버전이 로드돼요
**해결:**
1. Ctrl + Shift + R (강력한 새로고침)
2. 시크릿 모드로 열기
3. 브라우저 완전 종료 후 재시작

### 문제: 콘솔에 에러가 나와요
**해결:**
- F12 → Console 탭의 빨간색 메시지 캡처
- 메시지 내용 공유

### 문제: 카드가 렌더링 안 돼요
**콘솔에서 확인:**
```javascript
document.getElementById('assignment-cards-container')
```
- `null`이면 → HTML 버전이 오래된 것
- 객체가 나오면 → JavaScript 문제
