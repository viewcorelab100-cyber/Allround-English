# PDCA Plan: admin-mobile

## Feature Overview
- **Feature Name**: admin-mobile
- **Created**: 2026-04-08
- **Status**: Plan
- **Goal**: 관리자 페이지(admin.html)의 모바일 대응 가능 여부 분석 및 구현 계획 수립

---

## 1. 현재 상태 분석

### 1.1 admin.html 구조
- **총 6,795줄** (프로젝트 내 가장 큰 단일 파일)
- 레이아웃: `fixed 사이드바(w-64, 256px)` + `ml-64 메인 콘텐츠`
- 데스크탑 전용 설계: 모바일 브레이크포인트 대응 없음
- 모바일 헤더/햄버거 메뉴 없음

### 1.2 관리 섹션 (9개)
| 섹션 | 기능 | 모바일 난이도 |
|------|------|------------|
| Courses | 강의 CRUD, 레슨 관리, 드래그 정렬 | **높음** (테이블+모달+드래그) |
| Members | 회원 목록, 검색, 상세 | 중간 |
| Payments | 결제 내역, 필터 | 중간 |
| Coupons | 쿠폰 CRUD | 낮음 |
| Assignments | 과제 채점, 이미지 뷰어 | **높음** (이미지+채점 UI) |
| Orders & Shipping | 주문/배송 관리 | 중간 |
| QnA | 질문 답변 | 낮음 |
| Notifications | 알림톡 발송 | 낮음 |
| Popup Notices | 팝업 관리 | 낮음 |

### 1.3 모바일 대응의 핵심 장벽

| 장벽 | 설명 |
|------|------|
| **Fixed 사이드바** | `fixed left-0 w-64` → 모바일에서 전체 화면 차지 |
| **`ml-64` 콘텐츠** | 사이드바 너비만큼 마진 → 모바일에서 밀림 |
| **넓은 테이블** | 회원/결제/주문 등 테이블이 가로 스크롤 필요 |
| **복잡한 모달** | 강의 편집, 과제 채점 등 큰 모달 |
| **드래그 정렬** | 레슨 순서 변경 (터치 대응 필요) |
| **이미지 채점** | 학생 과제 이미지 확인 + 점수 입력 |
| **6,795줄 단일 파일** | 수정 시 사이드이펙트 위험 높음 |

## 2. 기술적 판단

### 2.1 결론: **가능하지만, 비용 대비 효과 낮음**

#### 가능한 이유
- Tailwind CSS 사용 중 → 반응형 유틸리티(`md:`, `lg:`) 추가 가능
- 사이드바를 햄버거 메뉴로 변환하는 패턴은 잘 알려져 있음
- 테이블은 카드 형식으로 변환 가능

#### 비용이 높은 이유
- 6,795줄 전체에 반응형 클래스 추가 필요
- 9개 섹션 각각의 테이블/폼/모달을 모바일 대응해야 함
- 드래그 정렬은 터치 라이브러리 별도 필요
- 과제 채점 UI(이미지 확인+점수 입력)는 모바일 UX 재설계 필요
- 테스트 범위가 매우 넓음 (9섹션 × 3 브레이크포인트)

#### 효과가 낮은 이유
- 관리자 = 원장(1명), 주로 데스크탑에서 사용
- 모바일에서 과제 채점, 강의 편집은 실용적이지 않음
- 관리 작업은 정밀한 클릭/입력이 필요 → 터치 UX 한계

### 2.2 대안 비교

| 방안 | 작업량 | 효과 | 추천 |
|------|--------|------|------|
| **A. 전체 반응형** | 매우 높음 (40h+) | 모든 기능 모바일 사용 가능 | ❌ |
| **B. 핵심 섹션만** | 중간 (15-20h) | Members/Orders/QnA만 모바일 | △ |
| **C. 읽기 전용 대시보드** | 낮음 (5-8h) | 현황 확인만 모바일 가능 | ⭕ |
| **D. 현상 유지** | 없음 | 데스크탑 전용 유지 | ⭕ |

## 3. 추천 방안

### 방안 C: 모바일 읽기 전용 대시보드 (추천)

모바일에서 **현황 확인**만 가능하게 하고, **편집/관리 작업은 데스크탑에서 유도**.

#### 모바일에서 가능하게 할 것
- 신규 회원 / 미결제 / 미채점 과제 수 확인
- 최근 주문 목록 확인
- QnA 미답변 목록 확인 및 간단 답변
- 알림톡 발송 (간단한 폼)

#### 모바일에서 하지 않을 것
- 강의/레슨 CRUD
- 과제 상세 채점 (이미지 확인)
- 쿠폰 관리
- 팝업 관리

#### 구현 방법
1. 사이드바: `768px 이하`에서 `display:none` → 햄버거 메뉴로 전환
2. `ml-64`: 모바일에서 `ml-0`
3. 대시보드 섹션 추가: 핵심 지표 카드 형식
4. 테이블: 모바일에서 카드 뷰로 전환 (Members, Orders만)
5. 복잡한 모달: 모바일에서 "데스크탑에서 확인하세요" 안내

#### 예상 작업량: 5-8시간
