# 🔒 강의 공개/비공개 시스템 가이드

## 📖 개요

이 시스템은 강의를 **공개** 또는 **비공개**로 설정할 수 있게 하여, 비공개 강의는 관리자가 승인한 사용자만 접근할 수 있도록 합니다.

---

## 🗄️ 데이터베이스 구조

### 1. `courses` 테이블 확장

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `visibility` | TEXT | 강의 공개 설정: `public`(전체 공개), `private`(비공개) |

### 2. `course_access_permissions` 테이블 (신규)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | UUID | 권한 ID (Primary Key) |
| `course_id` | UUID | 강의 ID (FK) |
| `user_id` | UUID | 사용자 ID (FK) |
| `granted_by` | UUID | 권한 부여한 관리자 ID |
| `granted_at` | TIMESTAMPTZ | 권한 부여 시간 |
| `notes` | TEXT | 권한 부여 사유 등 메모 |
| `created_at` | TIMESTAMPTZ | 생성 시간 |
| `updated_at` | TIMESTAMPTZ | 수정 시간 |

**제약조건:** `(course_id, user_id)` 조합은 UNIQUE (중복 권한 방지)

---

## 🔧 설치 및 설정

### 1. 데이터베이스 마이그레이션 실행

```bash
# Supabase SQL Editor에서 실행
create-course-visibility-system.sql
```

이 스크립트는 다음을 수행합니다:
- `courses.visibility` 컬럼 추가
- `course_access_permissions` 테이블 생성
- RLS (Row Level Security) 정책 설정
- 인덱스 생성
- 헬퍼 함수 생성:
  - `grant_course_access()` - 권한 부여
  - `revoke_course_access()` - 권한 삭제
  - `get_accessible_courses()` - 접근 가능한 강의 목록 조회

## 📋 주요 기능

### 1. 강의 생성/수정 시 공개 범위 설정

**위치:** `admin.html` > 강의 관리 > 강의 추가/수정

- **전체 공개 (public):** 모든 사용자가 강의를 보고 구매 가능
- **비공개 (private):** 관리자가 승인한 사용자만 강의를 보고 구매 가능

### 2. 비공개 강의 접근 권한 관리

**위치:** `admin.html` > 강의 관리 > 비공개 강의 옆 "권한 관리" 버튼

기능:
- **사용자 검색:** 이메일로 사용자 검색
- **권한 부여:** 특정 사용자에게 강의 접근 권한 부여
- **권한 목록:** 현재 접근 권한이 있는 사용자 목록 확인
- **권한 삭제:** 사용자의 접근 권한 삭제

### 3. 자동 필터링

- **강의 목록 페이지 (`courses.html`)**
  - 공개 강의: 모든 사용자에게 표시
  - 비공개 강의: 권한이 있는 사용자에게만 표시
  
- **강의 상세 페이지 (`course-detail.html`)**
  - 공개 강의: 모든 사용자 접근 가능
  - 비공개 강의: 권한이 없는 사용자는 접근 차단 메시지 표시

## 🔐 보안 정책 (RLS)

### 1. 강의 조회 권한

```sql
-- 공개 강의는 모두가 볼 수 있음
visibility = 'public' AND is_published = true

-- 비공개 강의는 권한이 있는 사용자만
visibility = 'private' AND 
is_published = true AND
EXISTS (권한 확인)

-- 관리자는 모든 강의 조회/관리 가능
```

### 2. 권한 테이블 접근

- **사용자:** 자신의 권한만 조회 가능
- **관리자:** 모든 권한 조회/관리 가능

## 🎯 사용 시나리오

### 시나리오 1: 신규 비공개 강의 생성

1. 관리자가 `admin.html`에서 "새 강의 추가" 클릭
2. 강의 정보 입력
3. "공개 범위"에서 **"비공개"** 선택
4. 강의 저장
5. 강의 목록에 🔒 승인제 배지 표시됨

### 시나리오 2: 사용자에게 비공개 강의 권한 부여

1. 관리자가 `admin.html` > 강의 관리로 이동
2. 비공개 강의 옆 **"권한 관리"** 버튼 클릭
3. 이메일로 사용자 검색 (예: `student@example.com`)
4. 검색 결과에서 **"권한 부여"** 버튼 클릭
5. 확인 후 권한 부여 완료
6. 해당 사용자는 이제 강의 목록에서 해당 강의를 볼 수 있음

### 시나리오 3: 권한 없는 사용자의 접근 시도

1. 사용자가 비공개 강의 URL을 직접 입력하여 접근 시도
2. 시스템이 자동으로 권한 체크
3. 권한이 없으면 다음 메시지 표시:
   ```
   🔒 비공개 강의입니다
   이 강의는 관리자가 승인한 사용자만 접근할 수 있습니다.
   접근 권한이 필요하시다면 관리자에게 문의해주세요.
   ```
4. "강의 목록으로 돌아가기" 버튼으로 리다이렉트

## 🔍 데이터베이스 함수

### 1. 권한 부여

```sql
SELECT grant_course_access(
    p_course_id := '강의-UUID',
    p_user_id := '사용자-UUID',
    p_notes := '권한 부여 사유 (선택)'
);
```

### 2. 권한 삭제

```sql
SELECT revoke_course_access(
    p_course_id := '강의-UUID',
    p_user_id := '사용자-UUID'
);
```

### 3. 접근 가능한 강의 목록 조회

```sql
SELECT * FROM get_accessible_courses('사용자-UUID');
```

## ✅ 체크리스트

### 강의 생성 시

- [ ] 강의 공개 범위 선택 (전체 공개 / 비공개)
- [ ] 비공개 강의의 경우, 설명에 접근 조건 명시 권장
- [ ] 강의 저장 후 목록에서 배지 확인

### 권한 관리 시

- [ ] 사용자 이메일 정확히 입력
- [ ] 권한 부여 사유 메모 (선택사항이지만 권장)
- [ ] 권한 부여 후 사용자에게 알림 (수동)

### 테스트 시

- [ ] 로그아웃 상태에서 비공개 강의 접근 차단 확인
- [ ] 권한 없는 사용자로 로그인하여 강의 목록에 표시되지 않는지 확인
- [ ] 권한 있는 사용자로 로그인하여 강의 접근 가능 확인
- [ ] 관리자는 모든 강의 조회/관리 가능 확인

## 🚀 향후 개선 사항

1. **자동 알림 시스템**
   - 권한 부여 시 사용자에게 이메일/문자 자동 발송

2. **대량 권한 부여**
   - CSV 파일로 여러 사용자에게 일괄 권한 부여

3. **권한 만료 기능**
   - 특정 기간 동안만 접근 가능하도록 설정

4. **권한 요청 시스템**
   - 사용자가 비공개 강의 권한을 직접 요청하고 관리자가 승인
